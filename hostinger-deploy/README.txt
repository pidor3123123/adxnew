ИНСТРУКЦИЯ ПО ЗАГРУЗКЕ НА HOSTINGER
====================================

Эта папка содержит только PHP часть проекта для загрузки на Hostinger.

ШАГ 1: ЗАГРУЗКА ФАЙЛОВ
----------------------
1. Заархивируйте ВСЕ файлы из этой папки в ZIP
2. Загрузите ZIP на Hostinger через File Manager или FTP
3. Распакуйте файлы в корневую папку вашего домена (обычно public_html)

ШАГ 2: НАСТРОЙКА БАЗЫ ДАННЫХ
-----------------------------
1. Войдите в панель Hostinger
2. Перейдите в раздел "Базы данных" -> "MySQL Databases"
3. Создайте новую базу данных (запомните имя БД, пользователя и пароль)
4. Откройте phpMyAdmin
5. Выберите созданную базу данных
6. Перейдите на вкладку "Импорт"
7. Выберите файл database.sql из этой папки
8. Нажмите "Вперёд" для импорта

ШАГ 3: НАСТРОЙКА ПЕРЕМЕННЫХ ОКРУЖЕНИЯ
--------------------------------------
1. Создайте файл .env в корневой папке (где находятся файлы проекта)
2. Скопируйте содержимое из .env.example (если есть) или создайте вручную
3. Заполните следующие переменные:

DB_HOST=localhost (или IP вашего MySQL сервера)
DB_NAME=имя_вашей_базы_данных
DB_USER=пользователь_базы_данных
DB_PASS=пароль_базы_данных
DB_CHARSET=utf8mb4

SUPABASE_URL=https://ваш-проект.supabase.co
SUPABASE_SERVICE_ROLE_KEY=ваш_ключ_supabase

WEBHOOK_URL=https://ваш-домен.com/api/webhook.php
WEBHOOK_SECRET=ваш_секретный_ключ

ALLOWED_ORIGINS=https://ваш-домен.com,https://админ-панель.домен.com

ШАГ 4: ПРОВЕРКА
---------------
1. Загрузите файл check_db.php на сервер
2. Откройте в браузере: https://ваш-домен.com/check_db.php
3. Проверьте, что подключение к БД работает

ВАЖНО:
- Не загружайте файл .env в публичный доступ
- Убедитесь, что права доступа к файлам установлены правильно (644 для файлов, 755 для папок)
- После настройки удалите check_db.php с сервера (это файл только для проверки)
