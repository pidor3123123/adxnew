# Готовый ZIP файл для деплоя на Hostinger

## Файл: `adx-finance-hostinger.zip`

Этот ZIP архив содержит все необходимые файлы для деплоя PHP API и фронтенда на Hostinger.

## Что внутри:

- ✅ Все PHP API файлы (`api/`), включая **`api/health.php`** (критически важно для мониторинга БД)
- ✅ Конфигурационные файлы (`config/`)
- ✅ HTML страницы (index.html, login.html, register.html и т.д.)
- ✅ CSS и JavaScript файлы
- ✅ База данных SQL (`database.sql`)
- ✅ Файл `.htaccess` для Apache
- ✅ Скрипты установки (`setup.php`, `check_db.php`, `check_health.php`)
- ✅ Документация

## Как использовать:

### Шаг 1: Загрузка на Hostinger

1. Войдите в панель Hostinger (hPanel)
2. Перейдите в **"Файлы"** → **"File Manager"**
3. Откройте папку `public_html`
4. Нажмите **"Загрузить"** (Upload)
5. Выберите файл `adx-finance-hostinger.zip`
6. Дождитесь загрузки
7. Выделите ZIP файл и нажмите **"Распаковать"** (Extract)
8. Удалите ZIP файл после распаковки

### Шаг 2: Настройка базы данных

1. Создайте базу данных MySQL в панели Hostinger
2. Импортируйте `database.sql` через phpMyAdmin
3. Запишите данные подключения

### Шаг 3: Создание .env файла

1. В File Manager создайте файл `.env` в корне `public_html`
2. Скопируйте содержимое из `env.example.txt` (в корне проекта)
3. Заполните реальными данными:

```env
DB_HOST=localhost
DB_NAME=ваше_имя_базы_данных
DB_USER=ваш_пользователь_базы_данных
DB_PASS=ваш_пароль_базы_данных
DB_CHARSET=utf8mb4

SUPABASE_URL=https://ваш-проект.supabase.co
SUPABASE_SERVICE_ROLE_KEY=ваш_ключ_supabase

WEBHOOK_URL=https://adx.finance/api/webhook.php
WEBHOOK_SECRET=сгенерируйте_сложный_секретный_ключ

ALLOWED_ORIGINS=https://adx.finance,https://adx.finance/admin
```

### Шаг 4: Проверка

1. **Проверка базы данных:**
   - Откройте `https://adx.finance/check_db.php`
   - Проверьте, что все проверки успешны
   - **ВАЖНО**: После проверки удалите `check_db.php`!

2. **Проверка Health Check API (критически важно для мониторинга БД):**
   - Откройте `https://adx.finance/api/health.php` в браузере
   - Должен вернуться JSON: `{"mysql":true,"supabase":true,"timestamp":"..."}`
   - Если видите 404 ошибку, проверьте, что файл `api/health.php` загружен на сервер
   - Альтернативно: откройте `https://adx.finance/check_health.php` для визуальной проверки
   - **ВАЖНО**: После проверки удалите `check_health.php`!

### Шаг 5: Проверка работы сайта

1. Откройте `https://adx.finance`
2. Попробуйте зарегистрироваться
3. Проверьте работу API

## Важные замечания:

- ⚠️ Убедитесь, что PHP версия 7.4 или выше
- ⚠️ Установите права доступа: файлы 644, папки 755
- ⚠️ Файл `.env` должен иметь права 600 (только владелец)
- ⚠️ После проверки удалите `check_db.php` и `check_health.php` с сервера
- ⚠️ **КРИТИЧНО**: Убедитесь, что файл `api/health.php` загружен на сервер (необходим для мониторинга БД)

## Устранение проблем

### Проблема: 404 ошибка при обращении к `/api/health.php`

**Причины:**
- Файл `api/health.php` не загружен на сервер
- Файл не был включен в ZIP при деплое
- Неправильные права доступа к файлу

**Решение:**
1. Проверьте наличие файла `api/health.php` в File Manager (должен быть в папке `public_html/api/`)
2. Если файла нет:
   - Загрузите файл `api/health.php` вручную через File Manager
   - Или пересоздайте ZIP архив, убедившись, что папка `api/` полностью включена
3. Проверьте права доступа: файл должен иметь права 644
4. Проверьте, что файл не пустой (размер должен быть > 0 байт)
5. После загрузки откройте `https://adx.finance/api/health.php` - должен вернуться JSON

### Проблема: Health Check возвращает ошибки MySQL/Supabase

**Решение:**
1. Проверьте настройки подключения в `.env` файле
2. Убедитесь, что базы данных созданы и доступны
3. Проверьте логи сервера для детальной информации об ошибках
